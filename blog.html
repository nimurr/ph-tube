Discuss the scope of var, let, and const ?

  1. Answer of Ver : Variables declared with var are function-scoped, meaning they are accessible within the function they are declared in, regardless of where they appear within that function.
  2. Answer of Let : Unlike var, let is not hoisted to the top of its block. This behavior prevents certain types of bugs and makes the code more predictable.
  3. Answer of Const : Note that this immutability applies to the binding itself, not necessarily the value it holds. For example, if a const variable holds an object, you can modify properties of the object, but you cannot assign a new object to the variable.


Tell us the use cases of null and undefined ?

  1. Answer of Null Vs Undefined : In many cases, null and undefined can be used interchangeably to represent absence of value, but it's a good practice to use them intentionally and consistently based on their semantics. If you're not sure which one to use, consider the context and whether the absence of a value is due to lack of initialization (undefined) or intentional omission (null).

What do you mean by REST API?

  1. Answer of Rest Api : RESTful APIs are widely used for building web services due to their simplicity, scalability, and compatibility with the existing infrastructure of the World Wide Web. They are commonly used to expose data and functionalities over the internet, enabling different applications to communicate and interact with each other seamlessly.
